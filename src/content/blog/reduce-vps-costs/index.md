Счёт за облако растёт незаметно: лишний VPS «на всякий случай», диск на 100 ГБ при занятых 8, staging, который крутится 24/7. В 2026 году FinOps — не роскошь, а необходимость. Ниже 12 способов, которые реально экономят 30–60% без деградации сервиса.

Методы проверены на проектах от одного VPS до десятков инстансов в AWS и GCP. Начните с аудита — без цифр оптимизировать нечего.

---

## 1. Правильный sizing: не переплачивайте за запас

Самая частая ошибка — VPS на 8 ГБ RAM при потреблении 800 МБ. Мониторьте реальное потребление через `htop`, Grafana или встроенную статистику провайдера.

- CPU idle >70% постоянно → downgrade на меньший тариф
- RAM usage <40% → уменьшите конфигурацию
- Проверяйте метрики за 2–4 недели, не за один день

Экономия: 40–70% от стоимости VPS при типичном overprovisioning.

---

## 2. Reserved instances и годовая оплата

У большинства провайдеров годовая или полугодовая оплата даёт скидку 15–40%. AWS Reserved Instances, GCP Committed Use Discounts — та же логика.

- Для стабильных production-сервисов — reserved
- Для эксперimentов и staging — pay-as-you-go
- Не покупайте reserved на 3 года, пока не уверены в архитектуре

---

## 3. Spot/preemptible instances для некритичных задач

Spot-инстансы дешевле на 60–90%, но могут быть прерваны. Подходят для:

- CI/CD runners (GitHub Actions self-hosted)
- Batch-обработка, рендеринг, ML-тренировки
- Staging-среды в рабочее время

Критичный production на spot не ставьте — только с fallback на on-demand.

---

## 4. Выключайте staging ночью и в выходные

Staging-копия production, работающая 24/7, — типичный «тихий» расход. Автоматизируйте:

- Cron: `0 20 * * 1-5 systemctl stop staging-app` (выключение в 20:00)
- Terraform/Ansible для поднятия утром
- Docker Compose `docker compose stop` вместо удаления

Экономия: ~65% стоимости staging-VPS при работе 8 часов в день.

---

## 5. Object Storage вместо дисков для бэкапов и медиа

Хранить бэкапы и uploads на VPS-диске — дорого и рискованно. S3-совместимое хранилище (AWS S3, MinIO, провайдерские object storage) стоит в 3–5 раз дешевле за ГБ.

- Бэкапы БД → S3 с lifecycle policy (удаление через 30 дней)
- Статика сайта → CDN + object storage
- Логи → Loki или S3, не локальный диск

---

## 6. CDN для статики

Отдавайте CSS, JS, изображения через Cloudflare (бесплатный тариф) или BunnyCDN. Нагрузка на VPS падает, можно взять меньший тариф.

- Кэширование на edge-серверах снижает bandwidth VPS
- Cloudflare Proxy скрывает IP и даёт базовый DDoS-защиту
- Для России — проверьте доступность CDN-провайдера

---

## 7. Контейнеризация: больше сервисов на одном VPS

Docker Compose позволяет запустить 5–7 сервисов на одном VPS вместо 5 отдельных серверов. Изоляция через контейнеры, общие ресурсы CPU/RAM.

- API + worker + Redis + PostgreSQL на одном 4 ГБ VPS
- Мониторинг (Uptime Kuma + Grafana) на отдельном минимальном VPS
- Не смешивайте production и staging на одном сервере без cgroups limits

---

## 8. Оптимизация базы данных

БД — частый потребитель ресурсов:

- Индексы на частые запросы — снижают CPU на 30–50%
- Connection pooling (PgBouncer) — меньше RAM на PostgreSQL
- Удаление старых данных по cron — таблицы не раздуваются
- Managed DB только если нет DBA; иначе self-hosted дешевле

---

## 9. Автомасштабирование вместо постоянного запаса

В облаках (AWS ASG, Kubernetes HPA) масштабируйте по нагрузке:

- 1 инстанс ночью, 3–5 в пик
- Scale-to-zero для serverless функций
- Horizontal scaling дешевле vertical при неравномерной нагрузке

На одном VPS autoscaling не работает — но можно мигрировать в облако при росте.

---

## 10. Аудит и удаление zombie-ресурсов

Раз в месяц проверяйте:

- Неиспользуемые VPS и floating IP (платятся даже без привязки)
- Snapshots старше 90 дней
- Load Balancers без backend'ов
- DNS-записи на удалённые серверы

Инструменты: AWS Cost Explorer, Infracost, ручной `terraform state list`. Один «забытый» VPS — минус 500–2000 ₽/мес.

---

## 11. Выбор провайдера и региона

Цены на идентичные конфигурации отличаются в 2–3 раза. Сравнивайте:

- vCPU/RAM/диск/NVMe vs SATA
- Включённый трафик (bandwidth overage дорог)
- Стоимость IP-адресов и бэкапов
- Локация: EU/US дешевле, но latency для RU-аудитории выше

Не гонитесь за самым дешёвым — downtime обходится дороже экономии 100 ₽.

---

## 12. Мониторинг расходов и бюджетные алерты

Настройте billing alerts:

- AWS Budgets → alert при 80% месячного лимита
- Telegram-бот от провайдера при превышении порога
- Еженедельный отчёт: cost by service, cost by tag

Без алертов счёт в 3 раза больше ожидаемого — стандартная история.

---

## Итог

Экономия на VPS и облаке — это sizing по реальным метрикам, reserved для стабильных сервисов, spot для batch, CDN для статики и регулярный аудит zombie-ресурсов. Комбинация пунктов 1, 4, 6 и 10 даёт 40–50% экономии на типичном стеке из 2–3 VPS без потери качества.

Начните с мониторинга потребления и billing alerts — через месяц увидите, куда уходят деньги. Для production выбирайте провайдера с прозрачным pricing и без скрытых платежей за трафик — Storm Cloud, например, даёт предсказуемые тарифы без сюрпризов в конце месяца.
